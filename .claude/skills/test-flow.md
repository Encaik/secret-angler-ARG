# test-flow skill — 验证游戏流程

测试指定阶段的游戏逻辑流程是否正确。

## 输入参数
- **目标阶段**：1-5
- **结局路径**（可选）：captured / lone_rescue / partial_victory / perfect_victory

## 输出格式

1. **流程步骤清单**：从阶段起点到终点的所有交互步骤
2. **状态变更日志**：每步操作后的 `arg_progress` 状态快照
3. **页面访问序列**：预期访问的 URL 路径列表
4. **谜题触发检查**：确认每个谜题在正确时机触发
5. **潜在逻辑漏洞报告**：
   - 是否存在跳过谜题的路径？
   - 是否存在死循环（需要某个线索但该线索需要前置）？
   - 结局触发条件是否正确排他？

## 验证方法

```
1. 打开浏览器隐身模式（清除 localStorage）
2. 从首页开始按步骤操作
3. 每步后检查 localStorage 中的 arg_progress
4. 对比实际状态与预期状态
5. 尝试"非常规路径"（直接访问隐藏 URL、跳过步骤等）
```

## 常见逻辑漏洞清单
- [ ] 隐藏页面是否可通过直接输入 URL 访问？（应该可以，但需要确认这是设计意图）
- [ ] 谜题答案是否可被暴力枚举？（SHA-256 保护）
- [ ] 结局条件是否存在重叠？（同一状态不可触发两个结局）
- [ ] 重置后的 localStorage 是否完全清空？
