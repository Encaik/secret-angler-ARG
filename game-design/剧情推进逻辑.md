#### 团伙能力边界（关键——决定故事可信度）

**团伙不擅长的事（硬伤来源）：**
- ❌ 历史学/考古学专业知识——成员没有学术背景，伪造的古迹描述必有硬伤
- ❌ 天文学/信号处理专业知识——外星信号类内容靠复制粘贴拼凑，经不起专业分析
- ❌ 地方志与民俗细节——全国撒网，无法对每个地点的历史文化做深入功课

**团伙擅长的事（他们真正的威胁所在）：**
- ✅ 网络运维与信息控管——搭建网站、SEO、伪造在线讨论氛围、得手后删除痕迹
- ✅ 目标筛选与行为分析——潜伏观察活跃用户发言模式、判断专业背景、评估下手难度
- ✅ 真实地点踩点——寻找本身就偏远、有天然"探秘价值"的真实地点作为诱饵（不需要造假景）
- ✅ 固定基地布景——地下基地的仿真场景搭建（仅限囚禁场所，不在野外暴露）
- ✅ 线下伏击与反追踪——在偏远地点提前埋伏，得手后迅速撤离，不留证据
- ✅ 暗网运营与加密通讯——匿名交易、加密货币收款

**这个能力边界意味着：**
- 导航站上的"秘境"描述对有专业背景的人来说漏洞百出
- 沈辞（古迹/历史爱好者）能发现历史描述的硬伤——这正是他最早起疑的原因之一
- 林屿（信号分析/天文爱好者）能发现"信号热点"的技术描述经不起推敲
- 但普通路人只会觉得"有趣"，不会深究——团伙的目标本来就不是专业人士

#### 单次作案全流程（从筛选到清理）

```
阶段1：潜伏观察（数周至数月）
  团伙成员伪装成普通用户（如"山野探秘人"），在社区中观察活跃用户。
  关注指标：发帖频率、探访偏好、装备水平、是否单独行动、社交圈大小。

阶段2：目标筛选
  找出符合条件的人：单独行动、去偏远地点、社交圈小、非本地人、无军方/警方背景。
  沈辞满足所有条件——这不是巧合，他是被选中的。

阶段3：匹配诱饵
  根据目标的兴趣推荐"秘境"——对沈辞这种古迹爱好者，推荐偏远古迹点位。
  注意：这些地点是真实存在的！团伙提前踩过点，确认够偏远、适合伏击。
  点位本身不需要是假的——它只需要够偏、能让目标独自前往。

阶段4：引导前往
  通过站内推送、私信、评论区暗示等方式，让目标"发现"这个点位并产生兴趣。
  关键：让目标觉得是"自己找到的"，而不是被引导的。

阶段5：线下伏击
  团伙提前到达点位附近埋伏。目标到达后实施绑架。
  不造假景——造假景太容易被发现，且成本高、风险大。
  野外唯一的痕迹是伏击者等待时留下的：篝火、食物包装、烟头、设备残片。

阶段6：清理痕迹
  得手后立即删除站内与该点位相关的推荐帖文、引导评论。
  目标最后发的帖文（如247号帖）会被保留——删除会引起其他用户警觉。
  但相关的"秘境推荐"、某些隐藏评论会被悄悄抹掉。

阶段7：消化与冷却
  受害者被转移到地下基地，用于暗网直播。
  接下来至少3个月内，该站点不再进行新的诱捕。
  如果社区中出现质疑声，团伙会控制舆论（删帖、沉帖、或派"山野探秘人"发"澄清"）。
```

#### 得手后清理痕迹——这本身就是线索

**团伙的操作：**
- 得手后，与该次诱捕相关的**点位推荐帖**、**引导性评论**会被悄悄删除或设为"仅管理员可见"
- 目标的最后一篇帖文会被**保留**（删除会引起其他用户警觉，反而暴露）
- 如果某个点位被多人质疑，团伙会直接删除整个推荐帖，让后来的探秘者"搜不到"

**玩家（林屿）能发现的疑点：**
- 某些"秘境推荐"的链接失效或内容被清空（但百度快照/搜索引擎缓存可能还有）
- 搜索某个关键词时，结果数量突然变少——有些内容被删了
- 不同时间截图的对比（如果玩家够细心）
- 沈辞247号帖下面的某些评论"不见了"（团伙可能删除了引导沈辞前往的评论）
- 搜索历史存档中某些帖文的作者最后登录时间异常（比如某个"推荐了贺兰山秘境"的人，在那之后就再没登录过——他可能也是受害者）

**在游戏中如何体现：**
- 搜索页可以设计"结果已被删除"的占位样式（模拟真实网站的删帖痕迹）
- 某些存档内容标注"该内容已被管理员移除"
- 沈辞私信中可能提到"之前有个推荐帖现在找不到了"
- 玩家搜索某个地名时，可能会出现"1条结果已被隐藏"的提示

#### 未得手的人——他们的吐槽就是线索

**不是所有去点位的人都会被伏击。** 团伙会判断：
- 来的人是不是目标？（不是目标→不下手，避免打草惊蛇）
- 来的人是不是一组人？（多人→不下手，风险太大）
- 现场条件是否合适？（天气差、有目击者→不下手）

**未得手的人回去后会：**
- 发帖吐槽"去了什么都没有"（如老废的废弃医院帖、阿浪的川西帖）
- 质疑导航站的信息来源
- 但这些帖子往往被团伙以"维护社区氛围"为由删除或沉底

**在现有游戏内容中已体现的角色：**
| 用户 | 帖子 | 性质 |
|------|------|------|
| 老废（废弃中） | 三个废弃医院假点位警告 | 去了但没被伏击（城市里不好下手，或者不是目标） |
| 阿浪（浪到没信号） | 川西秘境没找到 | 去了但没被伏击（背包客常结伴，不好下手） |

**这些帖子的叙事价值：**
- 让社区氛围更真实（不是只有沈辞发现了问题）
- 提供独立的情报源——玩家可以交叉对比多个人的发现
- 暗示"假点位"这个现象是系统性的，不是偶然
- 但普通用户止步于"信息有误"——只有沈辞/林屿这种有专业背景的人能看穿本质

#### 诱捕节奏与风险控制

**为什么不能频繁抓人：**
- 探秘/户外圈子不大，多个"探秘失踪"事件会被圈内人串联起来
- 家属报警+媒体关注=团伙暴露风险
- 暗网直播需要时间消化"库存"——每个受害者能支撑数周至数月的直播内容
- 偏远地点设伏需要踩点、布景、安排人手，周期至少1-2个月

**实际诱捕节奏（游戏时间线内）：**

```
2024年：通过"秘境徒步联盟"捕获 3人（户外圈，分散在全国不同地点，伪装成各自独立的山难/失联）
2025年：通过"星空观测者之家"捕获 2人 → 废弃该站
         启动"深空古迹探秘导航站"（新域名、新身份）
2026年5月：通过"深空古迹探秘导航站"捕获 沈辞（第1人）
           → 导航站计划再捕获 1-2人后废弃
```

**每两次诱捕之间的最短间隔：至少 3个月**
- 沈辞5月底失踪，下一次行动最早也是8月底
- 这意味着游戏"现在"（6月初），团伙正处于"消化沈辞+准备下一次"的阶段
- 如果玩家（林屿）在此期间表现出"要去实地探查"的迹象，他可能成为下一个目标

**受害者选择逻辑：**
- 优先选**单独行动**的人（组队去的人不好下手）
- 优先选去**偏远/信号差**地点的人（失踪解释更合理）
- 优先选**社交圈小**的人（少有人追问去向）
- 优先选**非本地人**（异地失踪跨区域追查难度大）
- ❌ 绝不选有军方/警方背景的人
- ❌ 绝不选有大量社交媒体粉丝的网红（失踪会引起舆论）

沈辞符合所有筛选条件：单独行动（虽然说是组队但实际一个人出发）、去偏远地区、大学生社交圈有限、非宁夏本地人。⚠️ 不是巧合——沈辞的"组队帖"（241号帖）当时确实有人响应，但团伙通过私信引导让他最终独自前往。

**暴露后的应对：**
- 一旦某个站点被多人举报或引起警方注意 → 立即废弃，删除敏感内容
- 团伙成员使用虚假身份注册，无法追踪到真人
- 服务器设在境外，域名使用匿名注册
- 如果事态严重，团伙会主动"自导自演"一波打假帖，伪装成社区自查自纠，掩护撤退

5. 这意味着：有人同时在伪造外星信号和古迹遗迹，把它们放在相同的地理位置
6. 结论：这不是某个人的恶作剧，而是一个有组织的系统性操作

### 4.6 历史硬伤线索的设计原则

作为"犯罪团伙不擅长学术"这一核心设定的体现，所有涉及具体古迹/历史描述的页面都应遵循以下规则：

**硬伤类型清单（创作时可参考）：**

| 硬伤类型 | 说明 | 示例 | 为何可信 |
|---------|------|------|---------|
| 年代错位 | 把不同朝代的东西混在一起 | 唐代烽燧描述用了清代砖瓦术语 | 考古专业学生一眼能认出 |
| 地理错误 | 虚构地名的地貌描述与实际不符 | 说某地有"喀斯特溶洞"但该地区是花岗岩地质 | 查地质图就能发现 |
| 文化混搭 | 把不同文化/文明的符号嫁接在一起 | 贺兰山岩画被描述成"类似玛雅图案" | 文化特征完全不同 |
| 术语滥用 | 用听起来"专业"但实际不存在的术语 | 编造"古越族星象石刻"这种不存在的概念 | 学术圈没有这个词 |
| 尺度错误 | 对遗迹规模的描述离谱 | "直径300米的古代祭坛"（实际不会有这么大） | 有常识的人会起疑 |
| 文献伪造 | 引用不存在的"古籍"或"县志" | "据《××县志》记载……"但该县志无此内容 | 如果真有人去查证会发现是编的 |

**页面实现时需注意：**
- 硬伤不应太明显（否则显得团伙太蠢），也不应太隐蔽（否则真玩家也发现不了）
- 最佳尺度：有相关知识的玩家能发现，普通玩家看到提示（如沈辞的动态/文章）后能理解
- 每个"秘境"页面建议包含 1-2 个硬伤，类型不重复
- 硬伤应埋藏在具体的描述细节中，而非标题或摘要中

## 八、叙事深度设计原则（核心——保证逻辑自洽与角色智商在线）

> 本章是编写和审查所有游戏内容时的**必须遵守的设计原则**。
> 每条原则都对应一个常见的叙事陷阱。违反任意一条都会让故事显得"里面的人很傻"。

### 原则一：犯罪团伙不是傻子

**叙事陷阱**：把反派写成"明明很容易暴露却没人发现"的蠢货。

**正确逻辑**：
- 团伙有专业分工（网络、场景、诱捕、暗网），但缺乏学术背景——这是他们唯一的能力短板
- 他们会控制诱捕频率（≥3个月间隔）、筛选目标（选社交圈小、单人行动、非本地人）、多站轮换（一个站抓够2-5人就废弃）
- 他们会在社区中安插眼线（"山野探秘人"等账号），监控是否有人怀疑、及时删除质疑帖
- 他们会主动"自导自演"打假帖来掩护——如果社区里质疑声太大，派个账号发帖"澄清"
- 暴露后会迅速废弃站点、删除数据、换身份重来

**检查清单**：新写的内容中，团伙的行为是否符合以上逻辑？如果有一个明显的漏洞而团伙没有堵上——为什么？

### 原则二：主角不是天才

**叙事陷阱**：林屿/沈辞"碰巧"发现了关键线索，或者一眼就看穿了一切。

**正确逻辑**：
- 沈辞能发现历史硬伤，是因为他真的是古迹爱好者——这不是"聪明"，这是"专业匹配"
- 林屿能发现信号伪造，是因为他每天都在做信号分析——这也不是"聪明"，这是"碰巧他的专业正好能识别这种伪造"
- 两人从各自专业角度独立发现异常 → 私信中分享各自的发现 → 交叉验证 → 确认系统性犯罪。这个链条每一步都需要时间（从4月到5月，近两个月）
- 他们之所以是目标，恰恰是因为他们太认真了——团伙的钓鱼内容是给"随便看看的人"设计的，但沈辞和林屿是"认真研究的人"

**正确叙事节奏**：
```
不是：第1天发现异常 → 第2天确认是犯罪 → 第3天去调查
而是：几周内积累疑点 → 交叉验证 → 几周后确认 → 准备后出发
```

**检查清单**：新写的发现/推理环节，角色是否利用了他们已有的专业知识？还是"灵光一闪"？

### 原则三：信息差是分层的

**叙事陷阱**：所有人都知道一样多，或者某个角色知道了不该知道的信息。

**正确逻辑**：不同角色能获得的信息深度不同，取决于他们的专业背景和访问权限。

| 信息层级 | 谁能知道 | 示例 |
|---------|---------|------|
| 公开层 | 所有访客 | 帖文表面内容、导航站推荐的点位 |
| 潜水层 | 注册用户 | 隐藏评论、会员可见版块 |
| 专业层 | 有相关背景的人 | 识别历史硬伤（沈辞）、识别信号伪造（林屿） |
| 私信层 | 对话双方 | 沈辞和林屿的坐标交叉验证（私信中讨论） |
| 加密层 | 特定登录者 | 沈辞私密草稿（仅shenci登录可见） |
| 幕后层 | 仅团伙成员 | 暗网直播、基地坐标、作案计划 |

**检查清单**：新写的内容中，每个角色获得的信息是否在其信息层级内？有没有角色"越级"知道了不该知道的事？

### 原则四：网友失踪要有合理解释

**叙事陷阱**：网上经常聊的人突然不见了，却没人觉得奇怪。

**正确逻辑**：
- 在探秘/户外圈子里，"失联几天"是正常的事（去偏远地区没信号）
- 沈辞出发前在247号帖中说"可能没信号，回来再跟大家分享"——这让他的失踪在最初几天看起来正常
- 失踪后社区的反应是有时间线的：第1天期待 → 第2-3天担忧 → 第4-5天有人尝试联系 → 第6-7天正式报警
- 团伙控制诱捕频率（≥3个月一次）意味着：圈子里不会同时出现多个失踪者，单个失踪更容易被解释为"意外"
- 如果频繁有人失踪——圈子里的人会串联起来，这是团伙极力避免的

**社区担忧的时间线（已在页面中体现）：**
```
5月26日（出发后1天）→ 山野探秘人留言"老哥什么时候回来？"（正常的关心）
5月27日（出发后2天）→ 星海漫步留言"有谁联系上地下三尺了吗？"（开始担忧）
5月28日（应归未归）   → 阿浪留言"希望只是没信号"（安慰，常见于户外圈）
5月29日-6月2日        → 可能的私下联系尝试（游戏中不直接体现）
6月3日（失踪约1周）   → 游戏"现在"，序幕中林屿开始系统性地寻找线索
```

**检查清单**：新写的失踪/失联相关内容，时间线是否合理？社区反应是否符合"逐步升级"的节奏？

### 原则五：钓鱼方案有生命周期

**叙事陷阱**：一个钓鱼网站运营了很久，抓了很多人，却没人发现。

**正确逻辑**：
- 每个钓鱼站点有明确的生命周期：启动 → 积累用户 → 筛选目标 → 诱捕2-5人 → 废弃
- 废弃的站点不会突然关闭（太可疑），而是停止更新、停止回复，自然地"死掉"
- 新站点使用全新身份（不同站长、不同域名、不同风格），避免被串联
- "深空古迹探秘导航站"目前已捕获1人（沈辞），接近废弃边缘
- 导航站上已有怀疑的声音（老废的警告帖、沈辞的文章）——如果再有1-2人质疑，团伙可能会提前废弃

**这解释了游戏中的一个潜在问题**：为什么导航站没有在沈辞开始怀疑的时候就把他"处理掉"？
- 答：因为沈辞的怀疑是渐进的，且他用了比较含蓄的方式表达（"关于这个站的一些疑虑"），没有直接"揭发"
- 团伙在评估：这个人是在随便质疑，还是真的发现了什么？前者可以忽略，后者必须处理
- 沈辞5月初发表的疑虑文章 + 他计划5月底实地探访贺兰山 → 团伙判断：他知道得太多了，而且他要去实地验证 → 决定在贺兰山对他下手
- 这也解释了为什么沈辞在247号帖中显示"被胁迫"的迹象（凌晨发帖、用老装备掩饰）——他可能在出发前就被威胁了

### 原则六：所有角色的行为都要有动机和约束

**叙事陷阱**：角色做了某件事"因为剧情需要"。

**正确逻辑**：每个角色的每个重要行动都应该有明确的动机和约束条件。

| 角色 | 行动 | 动机 | 约束 |
|------|------|------|------|
| 沈辞 | 5月25日独自去贺兰山 | 验证最后一个未确认的点位，收集证据 | 不能报警（没有确凿证据）、不能告诉太多人（怕打草惊蛇） |
| 沈辞 | 给林屿留加密信息 | 万一自己回不来，有人能继续查 | 不能明说（账号可能被监控） |
| 沈辞 | 凌晨发帖 | 可能是被胁迫后发的（或监控下发的） | 帖文内容表面正常但藏了暗语 |
| 林屿 | 沈辞失踪后没有直接去贺兰山 | 不知道沈辞具体去了哪个点位；需要先登录沈辞账号获取信息 | 没有头绪时贸然去找等于送死 |
| 林屿 | 在私信中讨论敏感内容 | 不在公开帖文中暴露怀疑 | 避免打草惊蛇——他不知道站里谁是团伙的人 |
| 团伙 | 不在沈辞质疑时就下手 | 评估威胁等级需要时间；频繁"处理"人会暴露 | 需要确认目标是否真的发现了核心秘密 |
| 团伙 | 让"山野探秘人"在社区活跃 | 监控社区舆论、引导话题、识别潜在目标 | 不能太明显——所以山野探秘人也发正常内容 |

**检查清单**：每个角色的每个重要行动，能不能清晰地说出"他为什么这么做"和"他不那么做的原因"？

### 原则七：三份文档必须同步更新

**叙事陷阱**：改了其中一个文档，忘了同步另外两个，导致各文档之间互相矛盾——按A文档走流程会卡住，按B文档写代码会发现逻辑不通。

**正确逻辑**：游戏内容（代码/页面）、剧情推进逻辑（本文档）、主线流程（`主线流程.md`）三者是**同一件事的三个视图**，任何一方的修改必须同步反映到另外两方。

```
游戏内容（代码/页面）  ←→  剧情推进逻辑（本文档）  ←→  主线流程（主线流程.md）
     │                          │                          │
     └──────────────────────────┴──────────────────────────┘
                    三者描述的是同一个游戏
                    任何修改必须三方同步
```

**三方各自的职责：**

| 文档 | 职责 | 典型内容 |
|------|------|---------|
| 游戏代码/页面 | 玩家实际接触到的内容 | 帖文文字、私信内容、谜题答案、页面交互 |
| 剧情推进逻辑（本文档） | 叙事逻辑是否自洽 | 角色动机、时间线一致性、信息层级、发现链条 |
| 主线流程（`主线流程.md`） | 流程是否可走通 | 谜题步骤、触发条件、密码/答案、阻塞点 |

**同步规则：**

1. **修改密码/答案** → 必须同步更新：游戏代码（哈希值/验证逻辑）+ 主线流程（答案记录）+ 本文档（相关线索描述）
2. **新增/修改帖文内容** → 必须同步更新：游戏代码（帖文数据）+ 主线流程（线索位置）+ 本文档（时间线、角色一致性）
3. **调整时间线** → 必须同步更新：游戏代码（帖文日期）+ 主线流程（触发时间窗口）+ 本文档（时间线全局一致性）
4. **新增/修改角色设定** → 必须同步更新：游戏代码（用户数据）+ 主线流程（角色相关谜题）+ 本文档（角色属性表）
5. **修改叙事逻辑（如发现链条、角色动机）** → 必须同步更新：游戏代码（页面内容体现新逻辑）+ 主线流程（流程步骤调整）+ 本文档（逻辑描述已更新）

**实际操作流程：**

```
修改任意一方时：
1. 在该文档中完成修改
2. 检查另外两方是否有对应内容需要同步
3. 在另外两方做对应修改
4. 在三方各自的更新日志中记录此次同步
5. 在本方修改的 commit message 中注明同步了哪两方
```

**常见违规场景（必须避免）：**

| 违规 | 后果 |
|------|------|
| 改了帖文日期但没更新本文档时间线 | 时间线逻辑检查失效，可能出现新的日期冲突 |
| 改了密码但没更新主线流程 | 主线流程记录的答案过期，开发/测试时按流程走不通 |
| 在本文档设计了新的发现链条，但游戏页面没更新 | 玩家在游戏中看不到新逻辑的体现，"设计得很深但玩家感受不到" |
| 主线流程标记某阻塞点"已解决"，但代码实际没改 | 流程文档失去参考价值，无法追踪真实进度 |

**commit 规范：**

修改涉及多方同步时，commit message 格式：
```
类型(范围): 简述

- 游戏内容：xxx 页面修改了 xxx
- 剧情逻辑：同步更新了 xxx
- 主线流程：同步更新了 xxx
```

**检查清单**：每次提交前，确认——如果改了三方中任意一方，另外两方有没有需要同步的内容？如果没有同步，是"确实不需要"还是"忘了"？
